# 하퍼세븐 요구사항 정리 (Product Manager Agent 산출물)

> 이 문서는 PM Agent 가 원본 요구사항을 MVP 범위로 재정리한 결과입니다.

## 1. 제품 목표

축구팀 **하퍼세븐** 운영자가 회원 정보 · 출석 · 경기 스탯 · 회비 납부 · 쿼터별 포메이션 배정을
한 곳에서 관리하는 웹앱. 초기에는 로컬 실행 가능한 MVP(홈페이지형 웹앱), 추후 모바일 앱 확장 가능.

## 2. MVP 범위 (이번 구현 포함)

| 영역 | 포함 기능 |
| --- | --- |
| 대시보드 | 기간 필터(월/분기/연/직접), 미납자 수·목록, 총 납부·예상·납부율, 출석/득점/어시 TOP5, 최근 경기 요약 |
| 회원 | 목록/추가/수정/삭제, 구분 변경, 나이·포지션·GK 입력, 엑셀 import/export |
| 회비 | 회원별 납부 현황, 월별 토글, 미납 자동 계산, 예상/실납부/미납, 환불 관리, 계좌 정보, 엑셀 export |
| 경기 | 경기 생성, 출석(참석/지각/부상/불참), 득점/어시/메모, 스탯 export |
| 포메이션 | 경기·템플릿 선택, 4쿼터 자동 배정, 셀렉트박스 수정, 규칙 위반 경고, **AI 채팅 조건 반영** |
| 통계 | 회원별 누적 출석/득점/어시, 기간별 랭킹 |

## 3. 추후 개선 기능 (Backlog)

- 드래그 앤 드롭 포메이션 편집 (MVP 는 셀렉트박스)
- 실제 DB(Supabase/Firebase) 연동 및 멀티 디바이스 동기화
- 모바일 앱(React Native) 확장
- 권한/로그인(관리자/회원 역할)
- 회비 자동 알림(카카오 알림톡 등)
- 경기 일정/캘린더, 출결 통계 그래프
- AI 채팅 대화 맥락을 ChatGPT 프로젝트에 실제 동기화(현재는 링크 연결)

## 4. 메뉴 구조

```
대시보드 / 회원 / 회비 / 경기 / 포메이션 / 통계
```

## 5. 기능 간 의존성

```
회원(Member)  ──┬─> 회비(Payment)        : 회원 구분 → 회비 정책 → 예상/미납 계산
                ├─> 경기(Match) 출석/스탯  : 회원 id 참조
                └─> 포메이션(Formation)    : 출석 + 회원 속성(나이/포지션/GK/구분)
경기(Match) ────> 포메이션                 : 참석자 명단 제공
포메이션 ───────> AI 채팅                  : 자연어 → ChatFormationRule → 알고리즘 입력
```

## 6. 운영 규칙 → 기능 변환 요약

| 운영 규칙 | 시스템 반영 |
| --- | --- |
| 회원 구분별 회비 | `lib/constants/feePolicy.ts` 상수로 정책화, 예상 회비 자동 계산 |
| 하반기 7월 말 납부 | 회비 화면 안내 문구 |
| 환불 2개월·1개월 2만원 | `calcRefundAmount()` + 환불 모달 |
| 인당 최소 3쿼터 | 포메이션 알고리즘 기본 규칙(`minGuaranteedQuarters=3`) |
| 13명 초과 우선순위 | `memberPriority.ts` + 알고리즘 차등 배정 |
| 고정 GK / GK 순환 | 알고리즘 GK 배정 로직 |
| 자연어 운영 지시 | AI 채팅 → ChatFormationRule |
